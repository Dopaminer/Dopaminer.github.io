0x00 PHP的序列化与反序列化介绍

1.序列化

PHP序列化，就是一个将各种类型的数据压缩并且按照一定格式存储的过程，所使用的函数是serialize()，实例如下：

对象的序列化格式一般就是：

O:<length>:"<class name>":<n>:{<field name 1><field value 1><field name 2><field value 2>...<field name n><field value n>}

O表示序列化的是对象，后面是对象所属的类的长度和名称，后面的数字是类的属性个数，然后大括号里就是每个属性的属性名，属性值。

这里我们观察到有两个问题，一是，序列化后的内容，为什么只有类的属性，而类的方法全部都扔掉了？这其实就是序列化的特性，只序列化属性而不序列化方法，所以说当我们在反序列化的时候，一定要保证在当前作用域环境下有该类存在。

第二个问题是，输出结果的属性，名称和长度都和原本的有些不同，比如代码里的是test1属性，输出的时候属性名变成了D0paminetest1，而且这一串字符的长度是13，给出来的却是15，于是把输出的内容放到一个文件里去看看。

用notepad打开发现，在与代码特异的部分前后还分别添加了两个空白字符，于是真相大白。

这其实是涉及到了php属性的访问权限问题，因为序列化要把所有的属性数据压缩，所以自然要保存原本的访问权限，所以就用了这种方式区别不同的访问权限。

public权限（默认） —- 原本代码是什么样他就是什么样，一点都没变。

protected权限 —- 序列化后属性名变成%00*%00属性名。

private权限 —- 序列化后属性名变成%00类名%00属性名。

0x01 PHP序列化和反序列化的作用

1.序列化压缩后变为一串字符，实现了传递数据的方便性。

2.为了可以长久的保存实例化的对象。因为php在执行完代码之后都会销毁创建的对象，有时候会造成许多不便，比如某一个页面刚好要用到上一次刚刚销毁的对象，就会非常麻烦，于是序列化实现了能把一个实例化的对象长久的存储在计算机磁盘上，要用的时候反序列化拿出来用就行了，方便快捷。

0x02 魔术方法(magic functions)

魔术方法就是在某些条件下会自动执行的函数，比如常见的构造，析构函数，就是典型的魔术方法。

为什么会有魔术方法：魔术方法是在需要实现一些功能，但是一般代码做不到或者很难做到的时候才能用。在命名自己的类方法时不能使用这些方法名，除非是想使用其魔术功能。魔术方法都是以两个下划线开头。

一些magic函数及用途：

__construct() ——– 当一个对象创建时触发

__destruct() ——– 当一个对象被销毁时触发

__toString() ——– 当类被当成字符串使用时触发

__sleep() ——– 使用serialize()时触发

__wakeup() ——– 使用unserialize()时触发

__get() ——– 当从不可访问的属性读取数据时触发

__call() ——– 在对象上下文中调用不可访问的方法时触发

0x03 反序列化漏洞

又叫做PHP对象注入漏洞。漏洞成因在于，代码中unserialize()函数接收的参数可控，而由于被传入的参数是已经序列化过后的，只保留了类的属性的对象，所以我们就可以利用篡改对象属性实现攻击。但是我们只能控制属性，而无法控制方法，相当于举步维艰，这时候魔法方法就派上用场了，因为魔法方法并不需要人工操控或者调用的代码才能触发，而是在一些特定的条件下自动触发，因此只要魔法方法中出现了一些我们可以利用的函数，就可以通过操控反序列化函数的参数，即操控对象属性来触发魔法方法，或者绕过魔法方法的触发，从而实现对函数的操控或绕过，达到攻击的目的。

0x04 利用phar://拓展PHP反序列化攻击面

在发现这一特性以前，php反序列化攻击的必要条件是——一定要有unserialize()函数，并且此函数的参数必须可控。但是在2017年hitcon发现这一0day以后，大大的拓宽了反序列化的供给面，给反序列化漏洞带来了很多新的可能，接下来就详解一下phar是什么以及如何利用phar://执行反序列化攻击。

1.phar是什么

PHP5.3之后支持了类似java的jar包，名为phar，可以用来将多个文件打包成一个phar文件，phar可以选择是否进行压缩，可以选择压缩的格式。

2.phar文件结构

phar由以下结构组成：

1.a stub

phar必须设置一个stub，可理解为一个标志，格式为xxx，前面内容不限，但必须以__HALT_COMPILER();?>来结尾，这部分的目的是让phar扩展识别这是一个标准的phar文件。

2.a manifest describing the contents

phar文件本质上是一种压缩文件，其中每个被压缩文件的权限、属性等信息都放在这部分。这部分还会以序列化的形式存储用户自定义的meta-data，有序列化就有反序列化，这是上述攻击手法最核心的地方。

3.the contents

被压缩的文件的内容。

4.signature

签名，放在文件末尾。
