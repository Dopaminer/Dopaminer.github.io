0x01 easy-function

知识点：

  1. php默认命名空间
  2. create_function()函数特性

解读一下，前两行那个??是php7的新特性，就相当于条件表达式

反正…大概在做的事情就是限制输入的变量action不能为空吧。

后面的正则，就是要求action的开头或者结尾有特殊字符又对函数名不造成影响，这里有个知识点，就是php的默认命名空间，引用p牛的话就是，php里默认命名空间是\，所有原生函数和类都在这个命名空间中。普通调用一个函数，如果直接写函数名function_name()调用，调用的时候其实相当于写了一个相对路径；而如果写\function_name() 这样调用函数，则其实是写了一个绝对路径。如果你在其他namespace里调用系统类，就必须写绝对路径这种写法。
所以这里刚好可以用\绕过正则。

其实整道题我们的目的就是最终实现RCE，所以后面构造的函数尤为关键，目前我能想到的符合所给的格式，也就是第一个参数是参数，第二个参数是可执行的内容的，也就是create_function()了。关于这个函数如何实现代码注入可以看这篇文章。![对create_function的介绍](/Code-breaking write-up/1.png)

0x02 pcrewaf

知识点： 利用PCRE回溯次数限制绕过正则检测

解读一下这个正则/<\?.*[(`;?>].*/is：/i表示不区分大小写，/s表示匹配任何不可见字符，如空格符，tab，换行符。.*表示匹配任意数量的任意字符。[(`;?>]方括号就是表示匹配方括号内任意一个字符就可以了。所以看这个架势其实这个正则匹配就是不允许传入的文件内有php代码。

如果上传的文件没有php代码，就会被临时存储，然后分配一个地址可以访问。

所以如何成功执行RCE的关键，就在于绕过正则。

这里涉及到的一个知识点，就是正则引擎，奉上p牛的一片文章。

常见的正则引擎有DFA和NFA两种，他们匹配的特点分别是：

  • DFA（确定性有限状态自动机）：从起始状态开始，一个字符一个字符地读取输入串，并根据正则来一步步确定至下一个转移状态，直到匹配不上或走完整个输入
  • NFA（非确定性有限状态自动机）：从起始状态开始，一个字符一个字符地读取输入串，并与正则表达式进行匹配，如果匹配不上，则进行回溯，尝试其他状态

NFA的执行过程中存在着回溯，所以性能会没有DFA好，但NFA的优点是支持更多功能，所以很多程序语言都在用NFA正则引擎，包括php使用的PCRE库。

假设输入的字符是<?php phpinfo();//aaaaa，来看一下正则匹配的过程：
![回溯过程](/Code-breaking write-up/3.png)可以看到，第四步的时候正则匹配到.*，既然是匹配任意多个字符，所以就直接匹配到了最后，即<?php phpinfo();//aaaaa，但是在.*后面还有[(`;?>]要匹配，最后的//aaaaa显然匹配不成功，所以就会开始回溯，也就是吐出一个a，然后依此类推回溯8次，到了12步，此时.*匹配的是<?php phpinfo()，[(`;?>]匹配的是;，这个结果满足了正则匹配的要求，于是不再回溯，13步继续往前执行，第二个.*就又匹配到了字符串结尾，结束匹配。

这个回溯的特点很有意思，而且也是这道题的关键，因为php为了防止正则表达式的拒绝服务攻击，给PCRE设置了回溯上限，可以通过var_dump(ini_get('pcre.backtrack_limit'));查看当前环境的回溯上限值，我这里目前是100万。

而当超过上限的时候，preg_match的返回值就不再是1或者0了，而是false，对于这道题来说刚好可以绕过正则的限制。

所以显而易见，我们应该构造一个payload，使他超出回溯次数，实现RCE执行。
